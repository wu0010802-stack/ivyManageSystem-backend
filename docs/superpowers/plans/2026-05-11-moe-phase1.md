# MOE Reporting Module — Phase 1: Schema Completion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add government-reporting structured fields to Student/Employee tables, create 4 new tables (one functional + three empty shells), wire up admin forms and a dashboard widget for disability certificate expiry alerts.

**Architecture:** Schema-only Phase. No exporters yet (those come in Phase 2-4). New `gov_moe` API module is created with two endpoints (disability documents CRUD + dashboard widget query); all other endpoints come later. Permission bits 1<<50 / 1<<51 are added but downstream modules will reuse them. Migration is reversible.

**Tech Stack:** FastAPI (Python), SQLAlchemy 2.x, Alembic, Pydantic v2, Vue 3 + Element Plus, axios, Vitest/pytest.

**Spec reference:** `docs/superpowers/specs/2026-05-11-moe-reporting-module-design.md` §4

### Test fixture pattern (IMPORTANT)

This codebase uses **self-contained per-file fixtures** (no shared
`authed_client_admin` / `sample_student` / `db_session` in `tests/conftest.py`).

The test snippets in this plan (Tasks 1, 5, 6, 7) illustrate intent and
expected assertions; when implementing, **wrap them with the self-contained
fixture pattern** following `tests/test_activity_academic_term.py`:

1. Create in-memory SQLite via `create_engine("sqlite:///<tmp>")` + `sessionmaker`
2. Monkey-patch `models.base._engine` and `_SessionFactory`
3. Call `Base.metadata.create_all(engine)` to create schema
4. Build minimal `FastAPI()` app including `auth_router` + the router under test
5. Seed an admin user via direct DB insert with `hash_password(...)`; login via `POST /auth/login` to get JWT; attach `Authorization: Bearer ...` header on subsequent requests
6. For teacher 403 tests: seed a second user with `role="teacher"` and 0 permission bits

The plan's permission-bit tests in Task 1 do NOT need a client (pure enum
assertions). Pydantic round-trip tests in Task 5 also do NOT need a client.
Only Task 6 and Task 7 endpoint tests need the TestClient pattern.

"""Dashboard widget endpoints for MOE module (Phase 1)."""
from datetime import date, timedelta
from typing import List, Optional

from fastapi import APIRouter, Depends, Query
from pydantic import BaseModel
from sqlalchemy.orm import Session

from models.database import get_db
from models.classroom import Student
from utils.permissions import Permission
from utils.auth import require_staff_permission

router = APIRouter(prefix="/dashboard")


class ExpiringStudentRow(BaseModel):
    id: int
    student_id: str
    name: str
    classroom_id: Optional[int]
    disability_cert_no: Optional[str]
    disability_cert_expiry: date
    days_remaining: int


class DisabilityExpiryResponse(BaseModel):
    total: int
    days_window: int
    students: List[ExpiringStudentRow]


@router.get("/disability-expiry", response_model=DisabilityExpiryResponse)
def disability_expiry_widget(
    days: int = Query(30, ge=1, le=365),
    db: Session = Depends(get_db),
    current_user: dict = Depends(require_staff_permission(Permission.GOV_REPORTS_VIEW)),
):
    today = date.today()
    end = today + timedelta(days=days)
    rows = (
        db.query(Student)
        .filter(Student.is_active == True)
        .filter(Student.disability_cert_expiry != None)
        .filter(Student.disability_cert_expiry >= today)
        .filter(Student.disability_cert_expiry <= end)
        .order_by(Student.disability_cert_expiry.asc())
        .all()
    )
    students = [
        ExpiringStudentRow(
            id=s.id,
            student_id=s.student_id,
            name=s.name,
            classroom_id=s.classroom_id,
            disability_cert_no=s.disability_cert_no,
            disability_cert_expiry=s.disability_cert_expiry,
            days_remaining=(s.disability_cert_expiry - today).days,
        )
        for s in rows
    ]
    return DisabilityExpiryResponse(
        total=len(students), days_window=days, students=students
    )
